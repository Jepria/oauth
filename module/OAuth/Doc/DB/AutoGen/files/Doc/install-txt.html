Общие замечания

Реализуется с помощью целей сборки модуля из make-целей Установка модуля в БД.  Для корректной установки должна быть установлена актуальная версия OMS (см.  Версия OMS-файлов).

Параметры установки

INSTALL_VERSIONУстанавливаемая версия модуля (Last при первоначальной установке или номер при обновлении)
LOAD_OPERATORIDЛогин и пароль оператора, выполняющего установку (формат: login/password)
LOAD_USERIDОсновной пользователь для установки в БД (формат: username/password@dbName)
LOAD_USERID2Основной пользователь для установки в резервную БД (формат: username/password@dbName)
LOAD_USERID3Пользователь для установки материализованных представлений в резервную БД (формат: username/password@dbName)
TO_USERNAMEИмя пользователя БД, которому выдаются права (формат: username)

Дополнительные параметры

SOURCE_DBLINKИмя линка к основной БД, доступного пользователю, указываемому в параметре LOAD_USERID3
SOURCE_SCHEMAСхема в основной БД, в которую установлены объекты модуля, используется при установке под LOAD_USERID3

Используемые макропеременные SQL*Plus (устанавливаются в SQL_DEFINE)

indexTablespaceТабличное пространство для индексов (значение по умолчанию определяется скриптом oms-set-indexTablespace.sql модуля OraMakeSystem)

Первоначальная установка в резервную БД

Порядок установки

  • выдача в основной БД прав пользователю, под которым работает линк из резервной БД
make grant LOAD_USERID=???/???@??? LOAD_OPERATORID=???/??? TO_USERNAME=??? GRANT_SCRIPT=link-user.sql
  • создание материализованных представлений в резервной БД
make install INSTALL_VERSION=Last LOAD_USERID3=???/???@??? LOAD_OPERATORID=???/??? SOURCE_DBLINK=??? SOURCE_SCHEMA=???
  • выдача прав на материализованные представления основному пользователю резервной БД
make grant LOAD_USERID3=???/???@??? LOAD_OPERATORID=???/??? TO_USERNAME=???
  • установка в основную схему резервной БД
make install INSTALL_VERSION=Last LOAD_USERID2=???/???@??? LOAD_OPERATORID=???/???
  • копирование ключа шифрования из основной в резервную БД

Для получения ключа в основной БД нужно выполнить:

SQL> select opt_option_list_t('OAuth').getString('CryptoKey') as key from dual;

Для установки ключа в основной схеме резервной БД нужно выполнить (заменив “<key>” на полученный ранее ключ):

begin
  opt_option_list_t('OAuth').setString('CryptoKey','<key>');
  commit;
end;

Выполнение тестов

Предварительные требования

  • установка модуля TestUtility;

Команда установки объектов для выполнения тестов

make install-test LOAD_USERID=???/???@??? LOAD_OPERATORID=???/???

Команда для выполнения всех тестов

make test LOAD_USERID=???/???@??? LOAD_OPERATORID=???/???

Если тесты выполняются из консоли с кодировкой CP1251, для корректного отображения кириллицы можно указать NLS_LANG:

make test LOAD_USERID=???/???@??? LOAD_OPERATORID=???/??? NLS_LANG=AMERICAN_CIS.CL8MSWIN1251

Замечания

  • для выполнения тестов в резервной БД используются аналогичные команды с указанием LOAD_USERID2 вместо LOAD_USERID;
